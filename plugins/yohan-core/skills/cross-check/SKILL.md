# cross-check — 적대적 교차검증

대상(코드/문서/계획)을 신뢰하지 말고 결함을 역추적한다.

## 절차
1. **주장 추출**: 대상이 암묵적으로 보장한다고 주장하는 것들을 나열.
2. **근거 확인**: 각 주장의 근거를 파일·라인·테스트로 확인. 근거 없으면 결함.
3. **반례 탐색**: 엣지케이스, 빈 값, 동시성, 권한, 인코딩(PS5.1 UTF-8), 경로(Windows) 등.
4. **보안/회귀**: 비밀 유출, 권한 확대, 기존 기능 깨짐 여부.
5. **판정**: 발견 결함을 심각도로 분류. 치명/높음이 있으면 **불통과** → 수정 요청.

## 출력
- 통과/불통과 + 결함 목록(위치·재현·제안 수정).
- 통과 시에만 다음 단계로.
