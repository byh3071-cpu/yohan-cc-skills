# /yohan-core:flow — 선순환 플로우 (승인 게이트 + 구현 검증)

작업: **$ARGUMENTS**

아래 순서를 그대로 따른다. 각 단계는 해당 서브에이전트에 위임한다.

## 1) 탐색 (explorer)
`explorer` 서브에이전트로 위임해 작업과 관련된 코드/문서/설정을 정찰한다.
- 추측 금지. 파일 경로·라인 근거만 수집.
- 산출물: 관련 파일 목록 + 현재 동작 요약 + 제약/리스크.

## 2) 설계 (planner)
`planner` 서브에이전트로 위임해 검증 가능한 단계로 분해한다.
- 작은 단위, 각 단계의 완료 기준 명시.
- 산출물: 단계별 체크리스트 + 영향 범위.

## 3) 🚦 승인 게이트 (사용자)
계획을 사용자에게 두괄식으로 제시하고 **명시적 승인을 받는다**.
- **승인 전에는 코드 작성 금지** (글로벌 규칙 "승인 전 코드 금지"). 이 게이트를 건너뛰지 않는다.
- 사용자가 수정을 요청하면 2)로 돌아가 반영 후 다시 제시.
- 승인이 떨어져야 4)로 넘어간다.

## 4) 구현
승인된 계획을 작은 검증 가능한 단위로 구현한다.
- 한 번에 한 슬라이스. 각 슬라이스는 독립적으로 확인 가능하게.

## 5) 검증 (critic) — 계획이 아니라 **구현된 diff**를 검증
`critic` 서브에이전트로 위임해 **실제 변경(diff)**을 적대적으로 교차검증한다(cross-check 스킬).
- 계획 준수 여부가 아니라 **실제 코드**의 반례·엣지케이스·보안·회귀를 역추적.
- 문제 발견 시 4)로 돌아가 수정. **문제 0이 될 때까지 4)↔5) 반복.**
- 되돌릴 수 없는 작업이 포함되면 PAT-003 안전장치를 확인.

## 6) 출시 (shipper)
`critic`가 **구현 검증**을 통과시키면 `shipper` 서브에이전트로 위임해 출시한다(ship-it 스킬).
- pre-commit 점검 → 두괄식 커밋 메시지 → push.
- 게이트 통과 시 `.claude/.gate-pass` 갱신.
