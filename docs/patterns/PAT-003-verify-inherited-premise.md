# PAT-003 — 상속·핸드오프 전제는 실측 재검증 후 신뢰

## 핵심 한 줄
핸드오프·문서의 "현재 X 상태다"는 SoT가 아니라 **스냅샷**이다. 그 전제로 계획하기 전에 **grep/read로 실값을 대조**하고, 틀리면 실측 기준으로 재작성 + 원본에 정정 노트 append.

## 실사례 (멀티모델 배분)
- 2026-07-02 핸드오프가 **"서브에이전트 model 필드 없어서 전부 Fable 단가($50/1M)로 상속 중"**이라 전제하고 6단계 계획을 세움.
- 실측(agents/*.md 4개 Read): **이미 tier 핀됨** — explorer=haiku, planner/critic/shipper=sonnet. 기본 지휘자도 Fable 아닌 **Opus 4.8**(plugin CLAUDE.md). $50 낭비는 애초에 없었음.
- 결과 계획이 **6단계 → planner·critic 2줄 + 독트린 정렬**로 축소. 실측 안 했으면 존재하지 않는 문제를 "해결"할 뻔함.

## 교훈 (역전파)
- 문서는 SoT 아님. "현재 X"는 코드가 SoT — 편집 전에 확인한다.
- 이전 세션 '나'의 핸드오프도 무검증 신뢰 금지(자기 출처라고 더 믿지 말 것).
- 전제가 틀리면 조용히 고치지 말고 원본에 정정 append — 다음 세션 보호.
