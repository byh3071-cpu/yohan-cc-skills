# plan-audit 자동 트리거 (C안) — 설계 검토

작성일: 2026-07-21
상태: **설계만. 구현 착수 전 실측 필요.** 코드 0줄.
선행: `plugins/yohan-core/skills/plan-audit/` (B안, v0.3.10에서 구현 완료)

## 문제

B안(`plan-audit` 스킬)은 사람이 호출해야 돈다. 호출을 잊으면 감사가 안 된다.
"계획 승인 직전"이라는 시점은 기계가 알 수 있으므로 자동 트리거가 가능해 보인다.

## 결론 먼저

**훅은 감사를 대신할 수 없다.** 훅은 PowerShell이라 "의도 정합"을 판정하지 못한다.
할 수 있는 건 **마커 확인**뿐이고, 그 마커를 만드는 건 결국 B안이다.
→ C안은 B안의 대안이 아니라 **B안 위에 얹는 트리거 레이어**다.

착수 게이트: **B안 실사용 데이터가 쌓인 뒤** 판단한다. 지금 만들면 근거 없이 마찰만 늘린다.

## 미검증 전제 2개 (착수 전 반드시 실측)

### 전제 1 — `PreToolUse` 매처가 `ExitPlanMode`에 발화하는가

[웹/외부] `https://code.claude.com/docs/en/hooks` 는 `PreToolUse`/`PostToolUse` 매처가 `tool_name`으로 필터한다고만 하고, `ExitPlanMode`가 매칭 대상인지는 **명시하지 않는다**. 추측 금지.

**실측 방법:** 넓은 매처로 프로브 훅을 걸고 `tool_name`을 로그에 남겨 `ExitPlanMode` 호출 시 발화하는지 확인.

**회피책:** 매처를 신뢰하지 말고 **스크립트 안에서 `tool_name`을 직접 확인**한다.
`plugins/yohan-core/hooks/critic-gate.ps1:5-12` 가 이미 같은 방어를 쓴다 — 주석에 "hooks.json의 if 매처가 적용 안 되는 환경 대비"라고 적혀 있다.

### 전제 2 — 훅이 계획 본문을 볼 수 있는가

`ExitPlanMode`의 `tool_input`에는 **계획 본문이 없다** (스키마상 deprecated `allowedPrompts`뿐).
따라서 훅은 `~/.claude/plans/*.md` 계획 파일을 직접 읽어야 한다. 파일 위치·명명 규칙이 버전 간 안정적인지도 함께 실측 대상.

## 설계 골자

| 요소 | 결정 | 근거 |
|---|---|---|
| 이벤트 | `PreToolUse` | 계획이 승인 다이얼로그로 가기 **전**에 끼어들어야 한다. `PostToolUse`는 이미 승인된 뒤라 늦다 |
| 판정 | `$env:TEMP\plan-audit\<sid>.pass` 마커 신선도 | 훅은 내용을 판정 못 한다. 감사가 돌았다는 증거만 본다 |
| 결정값 | `permissionDecision: "ask"` | **`deny` 안 쓴다.** `critic-gate.ps1:22` 선례 |
| 실패 정책 | fail-open (`exit 0`) | 어떤 실패도 세션을 막지 않는다. `detect-routing-miss.ps1:5` 규율 |
| 선별 | 계획 파일 휴리스틱(단계 수·언급 파일 수)으로 소규모는 skip | `ExitPlanMode`는 `git push`와 달리 고빈도다. 전부 걸면 S급까지 마찰 |

### `deny`를 안 쓰는 이유

강제해도 품질이 안 오른다. 훅이 막으면 **같은 세션의 같은 모델이 자기 계획을 자기가 고친다** — 자기확증 편향이 그대로다. 마찰만 늘고 검증은 그대로다.
`ask`로 두면 사람이 "지금은 넘어가자"를 고를 수 있고, 감사가 필요한 순간엔 리마인더로 작동한다.

## 선택 확장 — 텔레메트리 먼저

강제보다 **측정이 먼저**다. `detect-routing-miss.ps1`이 쓰는 순서를 그대로 따른다: 측정 → 축적 → 데이터 보고 룰 결정.

- `SessionEnd` 훅에서 계획 감사 실행 여부·지적 유형을 `~/.claude/.cache/*.jsonl`에 append
- **크로스레포 쓰기 회피** — 다른 레포 문서를 직접 고치면 dirty·충돌이 난다 (`detect-routing-miss.ps1:4`)
- 몇 주 뒤 "내 계획이 실제로 어디서 어긋나는가" 데이터로 루브릭 7항목을 조정

## 비범위

- 감사 자동 실행(사람 개입 0) — 감사 결과 판단은 사람 게이트로 남긴다

## 열린 질문

1. 계획 파일 경로 규칙이 CC 버전 간 안정적인가
2. 소규모 skip 휴리스틱의 임계값 — B안 실사용 로그가 있어야 정할 수 있다
3. 감사 통과 마커의 유효 기간 (critic-gate는 6시간)
